## Blocks "actually works" (must land before v0.1)

### Task A — Local-event drain into engine cycle

Today `engine::cycle::phase_delta_reconcile` only iterates remote items. Local-side changes
are invisible to the engine.

Two options:

1. **Add a `phase_local_uploads` step** that calls `LocalFs::scan`, inserts `FileEntry` rows
   with `sync_state = PendingUpload` for any path not yet tracked, and emits `Upload`
   decisions for entries whose `local` snapshot diverges from `synced`.
2. **Run FSEvents-driven detection in the scheduler**, which translates events into trigger
   batches and inserts the pending entries before invoking `run_cycle`.

Option 1 is the cleaner long-term shape (the engine owns reconcile). Option 2 is faster to
ship. Recommend Option 1 with a future-looking note that FSEvents push is an optimisation.

Verify the reconcile decision table still produces the right answers when `entry.local` is
populated but `entry.synced` is `None` (initial-sync case).

Commit: `feat(core/M12b): phase_local_uploads — detect untracked + diverged local files`

### Task B — Initial-sync state transition

`pair.add` mints a Pair with `status = Initializing`. Nothing transitions it to `Active`
after the first successful cycle. Add a guard in `run_cycle` (or in the scheduler post-cycle
hook) that flips `Initializing -> Active` when `outcome = Success` and the delta returned a
populated cursor.

Commit: `feat(core/M12b): Initializing -> Active transition on first successful cycle`

### Task C — Real-world OAuth audit

The PKCE flow has never been exercised against `login.microsoftonline.com`. Likely fixes:

- Redirect URI registration: docs say `http://localhost/callback` but the daemon binds an
  ephemeral port. Microsoft accepts `http://localhost:<port>` if `http://localhost` is
  registered without a port. Verify.
- Scope phrasing: `Files.ReadWrite offline_access User.Read`. Try the request once; surface
  any returned `invalid_scope` cleanly.
- Tenant routing: `common` should accept both MSA and AAD. Verify with a personal account
  and a work account.

Run the M9 `#[ignore]`'d end-to-end test (`m9_end_to_end`) against a real account; expand
its body to drive `account.login.begin -> manual paste -> account.login.await -> pair.add`.

Commit: `test(daemon/M12b): real-account end-to-end OAuth test (still #[ignore])`

### Task D — Per-connection subscription writer

The M10.1 `audit.tail` registers a subscription but no per-connection task reads from the
receiver. Implement:

1. Refactor `DispatchCtx` into a base struct + a per-connection wrapper (`ConnCtx`) that
   carries an `mpsc::Sender<JsonRpcNotification>`.
2. `ipc::server::handle_connection` creates the per-connection mpsc, spawns a writer task
   that pumps frames to the socket, and constructs the `ConnCtx` for each dispatch.
3. `audit.tail` (and future `pair.subscribe` / `conflict.subscribe`) hand the receiver to
   the writer.

Update all test fixtures (4 sites) to construct the new context shape.

Commit: `feat(daemon/M12b): per-connection subscription writer + ConnCtx refactor`

## Credible v0.1 polish

### Task E — Token refresh resilience

`VaultBackedTokenSource` caches but has no retry-on-network-blip and no handling for
`invalid_grant` (Microsoft revoked the token, user must re-authenticate). Add:

- Backoff retry on transient HTTP errors (Network, Transient).
- On `ReAuthRequired`, mark the account's pairs as `status = Errored` with
  `errored_reason = "re-auth required"` and emit an `account.re_auth_required` audit event.

Commit: `feat(daemon/M12b): token refresh retries + re-auth-required propagation`

### Task F — Scheduler-side case-collision detection

The `case_collision_rename_target` helper landed in M9.4 but no one calls it. In the
scheduler's pre-cycle local-scan step (Task A), detect collisions via `case_folds_equal` and
rename the local-side loser via `LocalFs::rename`, then record a `Conflict` row.

Commit: `feat(daemon/M12b): scheduler-side case-collision rename per 01-domain-model`

### Task G — Notifications + free-space + metered checks

- `InstanceConfig.notify` controls whether the daemon emits macOS user notifications. Wire
  via `osascript -e 'display notification ...'` for simplicity (no new crate dep).
- `InstanceConfig.min_free_gib`: before starting any upload, query the local volume's free
  space; if below the threshold, pause the pair with
  `errored_reason = "free disk below threshold"`.
- `InstanceConfig.allow_metered`: query `SCNetworkReachability` (or its async equivalent);
  if currently metered and the flag is false, skip the cycle.

Commit: `feat(daemon/M12b): notify + min_free_gib + allow_metered config respected`

### Task H — `service.upgrade.prepare` / `commit`

Two-phase upgrade: `prepare` drains in-flight cycles + closes the IPC socket politely;
`commit` execs the new binary. Document the upgrade flow in `docs/install/`.

Commit: `feat(daemon/M12b): service.upgrade.prepare and service.upgrade.commit`
