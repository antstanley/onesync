# onesync — Implementation Roadmap

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement each milestone's plan task-by-task. This roadmap is the dependency map; per-milestone plans hold the executable steps.

**Goal:** Deliver the system described in [`docs/spec/`](../spec/) — a macOS daemon and CLI that two-way-syncs a designated local folder with a designated OneDrive folder, written in Safe Rust.

**Architecture:** Hexagonal — pure-logic core surrounded by adapter crates that implement port traits, two binaries (`onesyncd`, `onesync`) wire core to adapters. Nine crates total: see [`docs/spec/02-architecture.md`](../spec/02-architecture.md).

**Tech Stack:** Rust 1.95.0, Tokio, `rusqlite` (bundled), `notify` (FSEvents), `reqwest` (rustls), `blake3`, `security-framework`, `clap`, `serde`, `thiserror`, `proptest`. Test runner: `cargo-nextest`. Coverage: `cargo-llvm-cov`. VCS: jujutsu (colocated git).

## M1 — Foundations

**Status:** Plan ready. See [`2026-05-11-m1-foundations.md`](2026-05-11-m1-foundations.md).

**Scope:**
- Cargo workspace, toolchain pinning, lint config, base CI commands.
- `onesync-protocol`: every canonical type from [`canonical-types.schema.json`](../spec/canonical-types.schema.json) as a Rust `struct`/`enum` with `serde` derive.
- `onesync-core`: port traits, `limits` module with every named const from spec 09, port-level error enums.
- `onesync-time`: `SystemClock` + `UlidGenerator` plus testing fakes.
- Schema-compliance test: fixture instances of each entity validate against `canonical-types.schema.json`.

**Exit criteria:**
- `cargo build` succeeds across all M1 crates.
- `cargo nextest run --workspace` passes.
- `cargo clippy --all-targets --all-features -- -D warnings` passes.
- `cargo fmt --all -- --check` passes.
- Every limit named in [`spec/09-development-guidelines.md`](../spec/09-development-guidelines.md) has a matching `pub const` with a unit-test asserting its type and bounds.
- Every entity in `canonical-types.schema.json` round-trips through `serde` and validates against the schema in test code.

## M2 — State + Local FS

**Status:** Plan to be written when M1 closes.

**Scope:**
- `onesync-state`: SQLite adapter via `rusqlite`, `refinery` migrations, all tables in [`spec/06-state-store.md`](../spec/06-state-store.md), `StateStore` port implementation, fakes.
- `onesync-fs-local`: `notify`-backed `FSEvents` watcher, BLAKE3 streaming hash, atomic temp-rename writes, NFC path discipline, fakes.

**Files (preview):**

```
crates/onesync-state/
  src/lib.rs, src/migrations/001_initial.sql, src/migrations/mod.rs,
  src/queries/{accounts.rs, pairs.rs, file_entries.rs, file_ops.rs,
              conflicts.rs, sync_runs.rs, audit.rs, config.rs},
  src/fakes.rs (in-memory StateStore)
  tests/integration_{schema, queries, retention, concurrency}.rs
crates/onesync-fs-local/
  src/lib.rs, src/watcher.rs, src/scan.rs, src/hash.rs,
  src/write.rs, src/path.rs, src/error.rs, src/fakes.rs
  tests/integration_{watcher, scan, write, rename, hash}.rs
```

**Key tests:**
- Migrations idempotent on second run; rollback denial works.
- `file_entries_dirty` returns only non-clean rows; partial index used.
- Retention compaction prunes expected rows; `VACUUM INTO` succeeds.
- FSEvents overflow flag surfaces correctly.
- Atomic write survives an injected crash between fsync and rename.
- BLAKE3 hash matches `b3sum` of the same file.
- NFC normalisation collapses canonically-equivalent paths.

**Exit criteria:** the same lint/test/format gates as M1, plus an integration test that scans a synthetic 1000-file tempdir, writes the result to a temp SQLite, and reads it back without divergence.

## M3 — Graph Adapter

**Status:** Plan to be written when M1 closes (M3 only depends on M1, can begin in parallel with M2).

**Scope:**
- `onesync-graph`: in-crate MSAL (auth-code + PKCE + refresh), `reqwest` with `rustls-tls`, `RemoteDrive` port implementation, delta pager, small-upload + upload-session paths, hash verification on download, error mapping per [`spec/04-onedrive-adapter.md`](../spec/04-onedrive-adapter.md).
- `onesync-keychain`: `security-framework` shim implementing `TokenVault`.

**Files (preview):**

```
crates/onesync-graph/
  src/lib.rs, src/auth/{pkce.rs, code_exchange.rs, refresh.rs},
  src/client.rs, src/delta.rs, src/upload/{small.rs, session.rs},
  src/download.rs, src/items.rs, src/throttle.rs, src/error.rs,
  src/fakes.rs (HTTP-mock-backed)
crates/onesync-keychain/
  src/lib.rs, src/macos.rs, src/fakes.rs
```

**Key tests:**
- PKCE verifier/challenge pair validates against RFC 7636.
- `id_token` claims parsed: `tid = 9188040d-…` ⇒ `AccountKind::Personal`.
- Delta pager handles `@odata.nextLink` and `@odata.deltaLink`; `resyncRequired` surfaces.
- Upload session resumes after `416`; chunk size is a multiple of 320 KiB.
- Download hash verification rejects `sha1`/`quickXor` mismatch.
- Error mapping covers every row of the [Graph error table](../spec/04-onedrive-adapter.md).
- Throttling: `Retry-After` honoured; `GRAPH_RPS_PER_ACCOUNT` token bucket gates requests.

**External test fixtures:** mock HTTP server (`wiremock`) for unit/integration; opt-in `e2e` tier against a real Microsoft Graph test tenant gated by env vars.

**Exit criteria:** lint/test/format gates pass; OAuth flow drives a live test-tenant sign-in successfully on a developer machine; offline tests cover every error mapping and adapter method.

## M4 — Sync Engine

**Status:** Plan to be written when M2 and M3 close.

**Scope:**
- Engine module inside `onesync-core` (no new crate — engine has no I/O; it composes ports).
- Reconciliation, conflict policy, retry/backoff, per-pair scheduling, op planning, observability hooks. All behaviour in [`spec/03-sync-engine.md`](../spec/03-sync-engine.md).

**Files (preview):**

```
crates/onesync-core/src/engine/
  mod.rs, cycle.rs, reconcile.rs, conflict.rs, retry.rs,
  planner.rs, scheduler.rs, executor.rs, observability.rs
  tests/property_{state_machine, reconcile, conflict_naming}.rs
  tests/integration_{cycle_clean, cycle_dirty, cycle_conflict,
                     retry_throttle, retry_auth, full_rescan}.rs
```

**Key tests (property-based with `proptest`):**
- `FileSyncState` machine: every transition documented in [01 — Domain model](../spec/01-domain-model.md) round-trips.
- Conflict rename: stem/suffix/host extraction stable; collision-bounded retries terminate.
- Retry backoff: total elapsed time across `RETRY_MAX_ATTEMPTS` falls within `[base·(2^N), base·(2^N) + jitter]`.
- Reconcile decision table: for every (synced, local, remote) combination, output is deterministic.

**Exit criteria:** engine drives a fake pair through ten synthetic cycles covering: clean, dirty-local, dirty-remote, conflict, retry-on-throttle, retry-on-auth, full-rescan, queue truncation, pair pause, pair error. All cycles deterministic under fake `Clock`/`IdGenerator`.

## M5 — Daemon + IPC

**Status:** Plan to be written when M4 closes.

**Scope:**
- `onesync-daemon` binary: process startup, port wiring, advisory lock, JSON-RPC server on Unix socket, subscription multiplexer, structured logging, signal handling, graceful shutdown.

**Files (preview):**

```
crates/onesync-daemon/
  src/main.rs, src/startup.rs, src/wiring.rs,
  src/ipc/{server.rs, framing.rs, dispatch.rs, subscriptions.rs},
  src/methods/{health.rs, account.rs, pair.rs, conflict.rs,
              audit.rs, run.rs, state.rs, service.rs},
  src/logging.rs, src/shutdown.rs, src/lock.rs
  tests/integration_{ipc_handshake, methods, subscriptions,
                     shutdown_drain, concurrent_clients}.rs
```

**Key tests:**
- Every method in [`spec/07-cli-and-ipc.md`](../spec/07-cli-and-ipc.md#methods) handled and round-trips through the socket.
- Subscription back-pressure: slow client doesn't block the daemon; subscription drops with audit when buffer overruns.
- Major-version handshake refuses mismatched CLI.
- Frame larger than `IPC_FRAME_MAX_BYTES` rejected with `PayloadTooLarge`.
- Two simultaneous daemons fail to acquire `onesync.lock`.

**Exit criteria:** lint/test/format gates pass; daemon starts under `cargo run`, accepts a hand-crafted JSON-RPC request via `nc -U`, returns a well-formed response.

**Exit criteria:** lint/test/format gates pass; `onesync service install` on a fresh macOS user account produces a running daemon and a working CLI within `INSTALL_TIMEOUT_S`. The repo at this point delivers the system in [`docs/spec/00-overview.md`](../spec/00-overview.md).

## Cross-cutting work (continuous, not a milestone)

The following appear in every milestone and are tracked per-milestone rather than as a separate phase:

- **Assertions:** every public function in `onesync-core` carries ≥ 2 meaningful `assert!`/`debug_assert!` statements ([gist guideline](https://gist.github.com/antstanley/5bdaa85e63427fadae1c58ae6db77c27)); enforced by `cargo xtask audit-asserts` from M2 onward.
- **Schema regeneration:** any change touching `canonical-types.schema.json` ships with the generated Rust types in `onesync-protocol`.
- **Audit events:** every limit reached emits `limit.<const_name>`; verified per-milestone.
- **CI:** GitHub Actions workflow added at the M2/M3 boundary (when there are multi-crate test commands to run); macOS runners only.
